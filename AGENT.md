# AGENT.md

이 문서는 이 저장소에서 작업하는 AI 코딩 에이전트/자동화 도구를 위한 실행 규칙입니다.

## 1. 작업 원칙

- 먼저 문제를 재현하고, 원인을 확인한 뒤 수정한다.
- 추정으로 수정하지 말고, 코드/로그/테스트 근거를 남긴다.
- 요청 범위를 벗어난 리팩터링은 별도 제안 후 진행한다.
- 기존 사용자 변경사항을 임의로 되돌리지 않는다.

## 2. 코드 변경 원칙

- 아키텍처 경계는 `docs/ARCHITECTURE.md`를 우선 따른다.
- 스타일/네이밍/리뷰 기준은 `docs/CODE_CONVENTION.md`를 따른다.
- FastAPI 라우터는 얇게 유지하고, 비즈니스 로직은 `application/domain`으로 이동한다.
- 새 기능에는 가능한 범위에서 테스트를 함께 추가한다.
- 하위 호환성에 영향이 있으면 변경 내용을 명시한다.

## 3. 안전 장치

- 비밀 정보(.env, API 키, 인증 토큰)를 로그/코드/문서에 노출하지 않는다.
- 파괴적 명령(`reset --hard`, 강제 삭제 등)은 명시적 지시 없이는 실행하지 않는다.
- 마이그레이션/데이터 변경 작업은 롤백 방법을 함께 제시한다.

## 4. 작업 절차 (권장)

1. 관련 코드와 문서를 확인하고 요구사항을 짧게 요약한다.
2. 수정 대상 파일과 영향 범위를 식별한다.
3. 최소 변경으로 구현한다.
4. 테스트/린트/정적 검사 등을 실행한다.
5. 변경 파일, 핵심 이유, 검증 결과를 보고한다.

권장 검증 명령:

- `pytest -q`
- `ruff check .`
- `black --check .`

## 5. 응답 형식

- 결과 먼저: 무엇을 바꿨는지 한 줄 요약
- 근거 다음: 왜 바꿨는지 핵심 이유
- 검증 마지막: 어떤 테스트/명령으로 확인했는지
- 실패 시: 막힌 원인, 시도한 내용, 필요한 입력을 명확히 제시

## 6. 문서 동기화 규칙

아래 변경이 발생하면 문서도 같이 업데이트한다.

- 계층/폴더 구조 변경 -> `docs/ARCHITECTURE.md`
- 네이밍/리뷰/테스트 규칙 변경 -> `docs/CODE_CONVENTION.md`
- 자동화 작업 방식 변경 -> `AGENT.md`
- Render 배포/환경 변수 규칙 변경 -> `docs/DEPLOY_RENDER.md`
